<h1 id="order_create">
order.create
<code>(string $merchantCode, object $items, object $address, object $info)</code>
</h1>

Create a new order. This operation will only be successful if the inventory is available and the unique_id is unique (if specified).

#### Parameters

<table class="table">
<thead><tr><th>order</th><th>description</th></tr></thead>
<tbody>
    <tr>
        <td>0</td>
        <td>string - Merchant code.</td>
    </tr>
    <tr>
        <td>1</td>
        <td>object - Order Items (key: Product SKU, value: Quantity)</td>
    </tr>
    <tr>
        <td>2</td>
        <td>object - Shipping Address (see "<a href="#address_properties">Address Properties</a>")</td>
    </tr>
    <tr>
        <td>3</td>
        <td>object - Order Additional Data (see below)</td>
    </tr>
</tbody>
</table>

#### Order Additional Data

<table class="table-striped">
<tbody>
    <tr>
        <th>unique_id</th>
        <td>
            <pre><code>{ "unique_id" : "100000005" }</code></pre>
            This field is optional and if not specified an auto-incrementing number will be assigned. Uniqueness based on this field
            is enforced so an error will be given if an order with the same unique_id already exists.
            <br/><br/>
            This 'unique_id' is the id used for other API calls such as "order.info".
        </td>
    </tr>
    <tr>
        <th>order_ref</th>
        <td>
            <pre><code>{ "order_ref" : "12345" }</code></pre>
            This field is optional and if not specified will be left blank. Uniqueness based on this field is not enforced (two orders
            may have the same order_ref value). If specified, this number will appear on packing slips in place of the unique_id.
        </td>
    </tr>
    <tr>
        <th>shipping_method</th>
        <td>
            <pre><code>{ "shipping_method" : "ups_01" }</code></pre>
            See the <a href="/doc/shipping-methods.html">Shipping Methods</a> document for a reference.
        </td>
    </tr>
    <tr>
        <th>custom_greeting</th>
        <td>
            <pre><code>{ "custom_greeting" : "Greetings!" }</code></pre>
            If not specified the configured default greeting will be used.
        </td>
    </tr>
    <tr>
        <th>note</th>
        <td>
            <pre><code>{ "note" : "This is a test." }</code></pre>
            The "Note" adds a status history to the order that only you and the warehouse staff may see.
            Adding a note will not affect the way the order is picked and packed.
        </td>
    </tr>
    <tr>
        <th>signature_required</th>
        <td>
            <pre><code>{ "signature_required" : "adult" }</code></pre>
            If "adult" then the package(s) will be shipped with the Adult Signature Required option.
        </td>
    </tr>
    <tr>
        <th>overbox</th>
        <td>
            <pre><code>{ "overbox" : 0 }</code></pre>
            Request overbox option.
        </td>
    </tr>
    <tr>
        <th>requested_ship_date</th>
        <td>
            <pre><code>{ "requested_ship_date" : "2014-07-29" }</code></pre>
            The requested ship date will determine if the merchant wishes for the item to be shipped on the same day or not.
            Whether or not this date becomes the Target Ship Date depends on the merchant's target ship date cutoff time and the time
            the order is placed.
        </td>
    </tr>
    <tr>
        <th>delayed_ship_date</th>
        <td>
            <pre><code>{ "delayed_ship_date" : "2014-07-29" }</code></pre>
            If specified, the order will be placed in "hold" status until this date.
        </td>
    </tr>
</tbody>
</table>

#### Return Value

An object with properties "order_id" and "unique_id".

#### Example Request

Create new order:

```javascript
{
    "jsonrpc" : 2.0,
    "id" : 1234,
    "method" : "call",
    "params" : [
        "be1c13ed4e03f0ed7f1e4053dfff9658",
        "sales_order.create",
        [
            "rsf",
            {
                "product2" : 5
            },
            {
                "firstname" : "Bill",
                "lastname" : "Gates",
                "company" : "Microsoft",
                "street1" : "11 Times Square",
                "city" : "New York",
                "region" : "NY",
                "postcode" : 10036,
                "country" : "US",
                "phone" : "212.245.2100"
            },
            {
                "order_ref" : 12345,
                "shipping_method" : "ups_03",
                "custom_greeting" : "Greeting text here",
                "note" : "Note text here",
                "signature_required" : false,
                "overbox" : false,
                "requested_ship_date" : "2014-07-28"
            }
        ]
    ]
}
```

#### Example Response

```javascript
{
    "jsonrpc" : 2.0,
    "id" : 1234,
    "error" : null,
    "result" : {
        "order_id" : 117,
        "unique_id" : "100000017"
    }
}
```


<h1 id="order_search">
order.search
<code>(null|object $filters, array $options = [], null|string|object $fields = [])</code>
</h1>

Retrieve list of orders by filters. Order data can be customized by specifying properties to retrieve.

#### Parameters

<table class="table">
<thead><tr><th>order</th><th>description</th></tr></thead>
<tbody>
    <tr>
        <td>0</td>
        <td><ul>
        <li>null - Retrieve list of all orders.</li>
        <li>object - Retrieve list of orders using specified <a href="/doc/search-filters.html" title="Search Filters">filters</a>.</li>
        </ul></td>
    </tr>
    <tr>
        <td>1</td>
        <td><ul>
        <li>null - No options will be applied.</li>
        <li>object - Apply specified <a href="/doc/search-options.html" title="Search Options">options</a>.</li>
        </ul></td>
    </tr>
    <tr>
        <td>2</td>
        <td><ul>
        <li>null - Retrieve only "order_id", "unique_id", and "order_ref" properties.</li>
        <li>string '*' - Predefined list of properties to retrieve. See "<a href="#order_properties">Order Properties</a>".</li>
        <li>object - Defined list of properties to retrieve in addition to "order_id", "unique_id", and "order_ref".</li>
        </ul></td>
    </tr>
</tbody>
</table>

#### Return Value

An array of objects. Each object will contain the specified (or default if no fields parameter was given) "<a href="#order_properties">Order Properties</a>".

#### Example Request

Get order status and order items for two order ids:

```javascript
{
    "jsonrpc" : 2.0,
    "id" : 1234,
    "method" : "call",
    "params" : [
        "be1c13ed4e03f0ed7f1e4053dfff9658",
        "order.search",
        [
            {
                "order_id" : {
                    "in" : [
                        114,
                        115
                    ]
                }
            },
            [
                "status",
                "items"
            ]
        ]
    ]
}
```
